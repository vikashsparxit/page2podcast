The Problem

AI coding tools stop at "write code, review code"

What's Missing

Real Delivery Has 10+ Activities

  • No threat modeling, pentesting, or security hardening
  • No architecture decision records or system design phase
  • No deployment strategy, observability, or post-deploy monitoring
  • No knowledge capture — lessons vanish between sessions
  • No visual output — everything stays in markdown

Our Approach

Understand First, Code Second

Vibe coding fails when AI guesses instead of knowing. This workflow scans your project, asks the right questions, and builds context before writing a single line.

  • Auto-detect your stack — languages, frameworks, cloud, quality tooling — no manual config
  • Deep codebase research — finds patterns, conventions, dependencies, and risks
  • Ask, don't assume — clarifies scope, acceptance criteria, and edge cases before planning
  • Progressive context — each phase feeds the next via structured artifacts
  • Learn & remember — retrospectives write lessons to CLAUDE.md; memory persists across sessions
/discover
Scans tsconfig, package.json, Dockerfile, .tf files, CI configs — maps your entire stack automatically
/research
Reads code, traces dependencies, identifies conventions — so generated code fits your project
/retro
Captures lessons learned and writes them to CLAUDE.md — future sessions start smarter

Semantic Context

Code Retrieval via claude-context

  • Hybrid search: BM25 keyword + dense vector semantic matching
  • Tree-sitter AST chunking — functions, classes, methods as units
  • Merkle tree incremental indexing — only re-embeds changed files
  • Local-first: Ollama embeddings + Docker Milvus storage
  • ~40% token savings — ranked chunks vs. iterative Glob/Grep rounds
  • Auto-enhances /research and /implement — finds code you didn’t know to search for

Context Engine

Code Intelligence Layer

  • Level 1: Repo Map + Symbol Index ≤3K tokens (type:name:file:line)
  • Level 1b: Dependency graph — Grep-based import tracing (repos ≥50 files)
  • Level 2: Targeted search via /retrieval MCP or Grep
  • Level 2b: 3-factor reranking — keyword (40%) + proximity (35%) + file-type (25%)
  • Level 3: Context Pack — ≤8 files (seeds + 1-hop deps + tests)
  • Smart activation — skips graph & reranking for small repos (<50 files)

Pipeline

L1: Repo Map + Symbol Index
func:validateToken:src/auth/mw.ts:15
class:AuthService:src/auth/svc.ts:8
        ↓ identify candidates
L1b: Dependency Graph
  mw.ts → imports [svc.ts, types.ts]
        ↓ enrich with relationships
L2: Search → L2b: Rerank → top-8
        ↓ expand + assemble
L3: Context Pack (≤8 files)
Zero external deps — all prompt-based (Glob + Grep). Cross-session cache via 01_DISCOVERY.md; intra-session via context window.
